1.主键：表主键约束，此列数据不能为空，不能重复，并且主键约束一个表中只能有一个
2.唯一约束：unique关键字可以建立字段为唯一约束,例如登录的用户名字段就是唯一约束
3.当表的主键字段为自增长时，要先删除自增长，才能删除主键
4.外键:两个表或多个表的外部外联关系。只有innodb引擎支持外键约束，且指向另一个表的主键或者唯一约束键
5.对于外键的插入必须在关联表中有一样的数据才能插入成功，否则失败(详细看下外键知识)
6.索引:本质相当于书籍目录，作用加快数据检索速度。缺点是消耗磁盘空间，降低插入和修改速度(需要更新索引)
7.order by rand()不建议使用,每次执行会对表刷新缓存
8.关系型数据库sql语句中的聚合函数不能直接用在条件语句中
9.很多数据库group by分组以后在查询的时候只能使用分组字段和聚合函数(select name,count(*) 总数 from student group by name)
10.where用于聚合前的条件筛选,having用于聚合后的条件筛选
11.多表查询只要保证2个表的字段有关系就行(例如:s.tid=t.id)
12.delete删除表会触发日志恢复操作和事件触发器,truncate不存在触发器。delete相当于一个记录一个记录删,truncate全部删除
13.查询过程中给让查询的字段有个别名显示的方法:select name n,age a from student
14.单项加密函数能加不能解,但能进行暴力破解(但已经有数学家给破解了,由密文推出明文)
15.数据库中的存储过程有利于数据库的分离,提高数据库表的安全(有封装的思想),且执行效率比直接写语句块 
16.在有些数据库存储过程的参数不允许写类型精度(例如simp_se(in name varchar),但mysql要求必须写)
17.创建触发器的触发事件为insert、update、delete语句事件
18.触发器就是级联动态操作,不需要一定有外键约束就可以实现级联操作(级联操作:对一张表的数据操作影响另一张表的数据)
19.mysql中函数有返回值,存储过程没有返回值
20.mysql定义的
21.①.用户变量：以"@"开始，形式为"@变量名",用户变量跟mysql客户端是绑定的，设置的变量，只对当前用户使用的客户端生效
   ②.全局变量：定义时，以如下两种形式出现，set GLOBAL 变量名  或者  set @@global.变量名对所有客户端生效。只有具有
      super权限才可以设置全局变量
   ③.会话变量：只对连接的客户端有效。
   ④.局部变量：作用范围在begin到end语句块之间。在该语句块里设置的变量declare语句专门用于定义局部变量。set语句是设
      置不同类型的变量，包括会话变量和全局变量
21.mysql事件功能默认是关闭的
22.mysql游标使用需要看书
23.记事本换行为'\r\n',editplus换行为'\n'
24.mysql导入导出出现乱码与字符编码有关
